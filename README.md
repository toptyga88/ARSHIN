## 💡 О проекте

**АРШИН** — это пайплайн для автоматического сбора и анализа данных о поверках счётчиков воды и тепла из государственного реестра [ФГИС Аршин](https://fgis.gost.ru/fundmetrology/cm/icsbase). Заменяет ручной Excel-справочник: данные обновляются автоматически, аналитика доступна в реальном времени через DataLens-дашборды.

### Зачем

|🔄|**Данные обновляются ежедневно** — ручной сбор устаревает за неделю      |
|-|-------------------------------------------------------------------------|
|🏷️|**23 бренда конкурентов** — у каждого десятки моделей и сотни поверителей|
|📈|**Динамика по месяцам** — кто растёт, кто падает, где и когда            |

## 🏗 Архитектура

Пайплайн из четырёх этапов. Каждый этап читает результат предыдущего и пишет в свою таблицу PostgreSQL.

```
┌────────────────┐    ┌──────────────────────────┐    ┌──────────────────┐    ┌──────────────────────────┐
│  ШАГ 0         │    │  ШАГ 1                   │    │  ШАГ 2           │    │  ШАГ 4                   │
│  dimbrand.py   │ ─▶ │  Обновлпервич.py         │ ─▶ │  енрич.py        │ ─▶ │  вольюмс.py              │
│                │    │                          │    │                  │    │                          │
│  dim_brands    │    │  manufacturers_verifiers │    │  dim_reference   │    │  production_volumes_exact│
└────────────────┘    └──────────────────────────┘    └──────────────────┘    └──────────────────────────┘
        │                        │                            │                          │
        │                        │                            │                          ▼
        └────────────────────────┴────────────────────────────┴──────────────▶  📊 Yandex DataLens
```

Сверху — оркестратор `запук.py`. Управляет очерёдностью, параметрами и флагами `RUN_*` для отдельных этапов.

## 🚀 Запуск

### Полный пайплайн (раз в полгода)

```bash
python запук.py
```

Запустит все этапы по очереди — от `dimbrand.py` до `вольюмс.py` для всех годов из `YEARS`.

### Только обновление объёмов (ежемесячно)

В файле `запук.py` поставить:

```python
RUN_DIMBRAND = False
RUN_PERVICH  = False
RUN_ENRICH   = False
RUN_VOLUMES  = True   # ← только этот шаг

YEARS = [2025]        # ← только текущий год
```

Дальше — `python запук.py`. Прогон будет в разы быстрее.

### Запуск отдельного шага

Любой скрипт можно запустить независимо:

```bash
python dimbrand.py
python Обновлпервич.py 2025 --pings 90 --positions 6 --threshold 70
python енрич.py
python вольюмс.py 2025
```

### Параметры в `запук.py`

```python
YEARS = [2023, 2024, 2025]   # за какие годы собирать данные

PINGS_TOTAL = 90              # пингов на рег.номер (умная выборка, шаг 1)
SAMPLE_POSITIONS = 6          # точек выборки по списку поверок
THRESHOLD = 70                # порог доли первичных, % → производитель
```

## 🗄 Структура БД

Четыре таблицы PostgreSQL связаны через ключевые поля.

### `dim_brands` — справочник брендов

|Колонка         |Тип   |Описание                        |
|----------------|------|--------------------------------|
|`id`            |SERIAL|PK                              |
|`бренд`         |TEXT  |UNIQUE — внутреннее название    |
|`поиск_mit`     |TEXT  |подстрока для поиска в API MIT  |
|`ключевые_слова`|TEXT  |варианты написания через запятую|
|`страна`        |TEXT  |из MIT detail                   |
|`город`         |TEXT  |из MIT detail                   |

### `manufacturers_verifiers` — производители по годам

|Колонка         |Тип    |Описание                             |
|----------------|-------|-------------------------------------|
|`id`            |SERIAL |PK                                   |
|`reg_number`    |TEXT   |UNIQUE *(reg_number, verifier, year)*|
|`verifier`      |TEXT   |организация-поверитель               |
|`year`          |INTEGER|UNIQUE *(reg_number, verifier, year)*|
|`primary_count` |INTEGER|первичных поверок в выборке          |
|`periodic_count`|INTEGER|периодических поверок                |
|`share_pct`     |REAL   |% первичных                          |
|`is_producer`   |BOOLEAN|TRUE если share_pct ≥ THRESHOLD      |

### `dim_reference` — характеристики моделей

|Колонка             |Тип    |Описание                                           |
|--------------------|-------|---------------------------------------------------|
|`id`                |SERIAL |PK                                                 |
|`рег_номер`         |TEXT   |UNIQUE *(рег_номер, категория)*                    |
|`бренд`             |TEXT   |из dim_brands                                      |
|`наименование_си`   |TEXT   |название счётчика                                  |
|`модификация`       |TEXT   |диаметры через запятую (15, 20)                    |
|`категория`         |TEXT   |UNIQUE — бытовые/домовые/промышленные/теплосчётчики|
|`теплосчётчик`      |BOOLEAN|определяется по слову в наименовании               |
|`ручное_исправление`|TEXT   |заполняется человеком если автоматика не справилась|

### `production_volumes_exact` — объёмы по месяцам

|Колонка                 |Тип    |Описание                                                                                             |
|------------------------|-------|-----------------------------------------------------------------------------------------------------|
|`id`                    |SERIAL |PK                                                                                                   |
|`рег_номер`             |TEXT   |UNIQUE *(рег_номер, год, месяц, категория, поверитель)*                                              |
|`организация_поверитель`|TEXT   |UNIQUE                                                                                               |
|`год`                   |INTEGER|UNIQUE                                                                                               |
|`месяц`                 |INTEGER|UNIQUE — 1..12                                                                                       |
|`категория`             |TEXT   |UNIQUE — четыре возможные категории                                                                  |
|`количество`            |INTEGER|число поверок                                                                                        |
|`точность`              |TEXT   |`точно` / `пропорция_из_выборки` / `теплосчётчик` / `одна_категория` / `не_определено` / `из_выборки`|

## 🌐 Источник данных

Данные берутся из публичного API ФГИС Аршин.

### MIT — справочник моделей

```
GET https://fgis.gost.ru/fundmetrology/eapi/mit?search=<подстрока>
GET https://fgis.gost.ru/fundmetrology/eapi/mit/<uuid>
```

Что используется: `number` (рег.номер), `title`, `manufacturers`, `mit_uuid`, `notation`, страна и город.

### VRI — записи о поверках

```
GET https://fgis.gost.ru/fundmetrology/eapi/vri?mit_number=<X>&year=<Y>&...
GET https://fgis.gost.ru/fundmetrology/eapi/vri/<id>
```

Что используется: `mit_number`, `verification_date`, `org_title`, `mi_modification`, `vriType` (`'1'` — первичная, `'2'` — периодическая), `count`.

### Особенности и грабли

|🐛|**HTTP 500 на помесячных запросах** — параметр `verification_date_start/end` иногда падает. Решение: страховочная выборка за весь год.|
|-|--------------------------------------------------------------------------------------------------------------------------------------|
|📏|**Лимит API** — параметр `start ≤ 99 999` (≤ 100k записей за серию). Для крупных производителей решается умной выборкой.              |
|🤔|**Нет признака «производитель»** — определяется эвристически по доле первичных поверок (≥ 70%).                                       |
